# Dashboard 2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a server-first `/dashboard` based on the real `sales`, `bills`, and `parcels` API behavior, with period filtering, paginated aggregation, clickable KPIs, analytical tables, and detail drawers.

**Architecture:** Use a server-rendered App Router page that loads all required pages for the selected date range, normalizes raw API responses into stable domain models, computes analytics in pure functions, and passes a compact view model to a client component for interaction. Keep enrichment from `patients` and `financial-categories` optional so the dashboard remains usable when those endpoints fail.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Tailwind CSS 4, Recharts, Node test runner.
